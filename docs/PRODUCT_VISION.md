## 2. VISÃO DO PRODUTO

**SERVIX** é um marketplace de serviços sob demanda e agendados que conecta clientes a profissionais verificados de forma rápida, segura e confiável — com a mesma simplicidade do Uber para contratar uma diarista, um eletricista, uma babá ou qualquer profissional de serviços domésticos e pessoais.

### Missão
Democratizar o acesso a serviços de qualidade, criando a plataforma de referência global para contratação de profissionais com confiança, transparência e eficiência.

### Visão de Longo Prazo
Ser o maior marketplace de serviços da América Latina nos primeiros 3 anos, expandindo para Europa e África Lusófona, processando +R$1 bilhão em GMV anual até o ano 5, com valuation para rodada Série A/B ou exit estratégico.

## 6. JORNADA DO USUÁRIO

### Jornada do Cliente — Contratação Imediata

```
1. DESCOBERTA
   App instalado → Onboarding (30s) → Perfil criado

2. BUSCA
   Home → Seleciona categoria → Lista de profissionais
   (filtro: avaliação, preço, distância, disponibilidade)

3. ESCOLHA
   Perfil do profissional → Verificações visíveis → Avaliações
   → Preço estimado → "Contratar agora"

4. CHECKOUT (máx 2 telas)
   Endereço → Data/Hora → Breakdown de preço
   → Método de pagamento → "Confirmar e Pagar"

5. ACOMPANHAMENTO
   Status do pedido → Chat → Notificações
   → Profissional chegou → Serviço em andamento → Concluído

6. PÓS-SERVIÇO
   Avaliação → Opção de recontratar → Favoritar
   → Sugestão de recorrência
```

### Jornada do Profissional — Onboarding e Primeiro Trabalho

```
1. CADASTRO
   Nome, e-mail, telefone → OTP → Tipo de conta: Profissional

2. ONBOARDING (5-7min)
   Foto do rosto → Documento (CPF/RG) → Categorias de serviço
   → Experiência → Preço base → Raio de atendimento
   → Disponibilidade → Dados bancários

3. VERIFICAÇÃO
   Análise automática + Manual se necessário
   → Aprovado: perfil ativo com "Verificado"

4. PRIMEIRO TRABALHO
   Pedido chega → Notificação → Aceitar/Recusar
   → Rota até cliente → Marcar chegada
   → Concluir → Avaliar cliente

5. RECEBIMENTO
   Saldo atualizado → Solicitação de saque
   → Pagamento em D+1 (configurável por país)
```

## 7. COMO OS 5 PILARES FORAM INCORPORADOS

### Pilar 1: Marketplace + Recorrência
- Entidade `RecurringServicePlan` é cidadã de primeira classe na modelagem
- Checkout oferece opção "uma vez" ou "recorrente" antes de confirmar
- Cobrança recorrente preparada via `payment_schedules`
- Histórico de prestadores favoritos alimenta sugestões de recorrência

### Pilar 2: Forte Camada de Confiança
- `TrustScore` calculado automaticamente por múltiplos fatores
- `ProviderVerification` com status rastreável e auditável
- `ServiceGuarantee` vinculada a cada pedido concluído
- Selos visuais na UI comunicam verificação em cada ponto

### Pilar 3: Motor Configurável por País
- Entidade `Country` com todas as configurações por mercado
- `FeatureFlag` por país permite ativar/desativar funcionalidades
- Gateways de pagamento intercambiáveis via `PaymentGatewayConfig`
- Comissões, taxas e regras fiscais configuráveis por `country_id`

### Pilar 4: Data e Inteligência
- `AnalyticsEvent` captura cada ação do usuário
- `MatchingScore` alimenta ranking de profissionais
- `PricingSignal` coleta dados para sugestão de preço futuro
- Estrutura pronta para ML pipeline externo consumir dados

### Pilar 5: Receita Diversificada
- 8 fluxos de receita implementados ou preparados no MVP
- `MonetizationLedger` rastreia cada centavo de cada fonte
- `SponsoredSlot` pronto para ativação sem refatoração
- `PremiumSubscription` com feature flags controláveis

## 8. ESTRATÉGIA DE VALUATION

### Alavancas de Valuation

| Alavanca | Impacto | Prazo |
|----------|---------|-------|
| GMV crescente | Múltiplo de receita | Contínuo |
| Take rate otimizado | Rentabilidade | 6-12m |
| Recorrência alta | Previsibilidade de receita | 3-6m |
| NPS > 70 | Retenção e word-of-mouth | 6m |
| Expansão geográfica | TAM maior | 12-18m |
| Data proprietary | Moat competitivo | 12m+ |
| Profissionais verificados escassos | Barreiras de entrada | 6m |

### Métricas que Investidores Olham

```
GMV (Gross Merchandise Value)
Take Rate = Receita Plataforma / GMV
LTV / CAC > 3x
Monthly Active Providers (MAP)
Monthly Active Clients (MAC)
Retention D30, D60, D90
Recurrence Rate (% pedidos recorrentes)
Net Revenue Retention (NRR)
Booking-to-Completion Rate
Time to First Booking (TTFB)
```

### Funcionalidades que Geram Moat
1. **Dados de reputação** — anos de histórico de avaliações não são replicáveis
2. **Rede de profissionais verificados** — KYC tem custo e tempo
3. **Histórico de preços regionais** — pricing intelligence proprietário
4. **Algoritmo de matching** — treinado em dados exclusivos da plataforma
5. **Relacionamentos de recorrência** — clientes fidelizados são difíceis de migrar

### Tese de Exit
- **Aquisição estratégica**: OLX, QuintoAndar, iFood, Rappi buscando expansão vertical
- **PE/Growth**: Fundo que busca consolidar marketplaces LatAm
- **IPO regional**: Com GMV > R$500M, múltiplo SaaS-like por recorrência
